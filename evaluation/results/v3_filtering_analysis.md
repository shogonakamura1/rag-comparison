## 結論

### v3a（翻訳なし + フィルタリング）が最良
- v1から+0.2ポイント改善
- Q8のストリーミングがtop_k拡大で回復
- Q2のコンテキストウィンドウ情報がやや改善
- 翻訳のAPI呼び出しが不要なため、**レイテンシが低く、コストも半分**

### 翻訳は現時点では不要
- multilingual-e5-largeは日本語クエリ×英語文書のクロスリンガル検索に対応
- 翻訳を追加してもスコア向上なし、むしろQ2で微減
- 翻訳のAPI呼び出し分のコストとレイテンシがデメリット

### 推奨構成
```
Embedding: multilingual-e5-large (翻訳不要)
top_k: 8
距離閾値: 0.45
翻訳: OFF
```
