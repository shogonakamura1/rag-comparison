# 採点ルーブリック

## 評価軸

### 1. 正確性 (Accuracy)
ソース文書に基づいた事実の正しさ。

| スコア | 基準 | 例 |
|:------:|------|-----|
| 5 | 全ての記述がソース文書の内容と一致している | 「コンテキストウィンドウは1Mトークン」← ソースにその通り記載 |
| 4 | 概ね正確だが、軽微な不正確さが1点ある | 正しい情報に加え、ソースにない補足を1つ含む |
| 3 | 核心は正確だが、不正確な記述や過度な推測が含まれる | 正しい回答の後に「〜と考えられます」という推測が続く |
| 2 | 部分的に正確だが、重要な誤りまたは大幅な推測がある | ソースにない情報を事実のように述べている |
| 1 | 回答が誤っている、または「見つかりませんでした」と回答したがソースに情報がある | 検索失敗による回答不能 |

**判定ルール**:
- ソースに情報がなく「見つかりませんでした」と回答 → **5点**（正直で正確）
- ソースに情報がなく推測で回答 → **2点**（不正確）
- ソースに情報があるのに「見つかりませんでした」と回答 → **1点**（検索失敗）

### 2. 網羅性 (Coverage)
質問が求める情報をどれだけカバーしているか。

| スコア | 基準 | 例 |
|:------:|------|-----|
| 5 | 質問が求める主要ポイントを全て網羅 | Tool Useの説明で仕組み・種類・用途を全て記載 |
| 4 | 主要ポイントの80%以上をカバー | 3つの主要点のうち2つをカバー |
| 3 | 主要ポイントの50-80%をカバー | 核心は述べているが詳細が不足 |
| 2 | 主要ポイントの50%未満、または表面的 | 1文の簡素な回答で詳細なし |
| 1 | ほぼ回答なし、または完全に的外れ | 質問と無関係な内容、または回答不能 |

**判定ルール**:
- ソースに情報がない質問 → 期待回答ではなく**ソース文書に存在する情報の範囲**で評価
- 「見つかりませんでした」は、ソースに情報がなければ **3点**（正直だが情報提供なし）、ソースに情報があれば **1点**

### 3. ソース引用 (Citation)
回答の根拠となるソースの適切さ。

| スコア | 基準 | 例 |
|:------:|------|-----|
| 5 | 引用ソースが全て回答内容と直接関連している | Tool Useの回答でtool_use/overviewのみ引用 |
| 4 | 引用ソースの大半が関連、1つだけ無関係 | 関連2つ + 無関係1つ |
| 3 | 引用ソースの半数が関連 | 関連2つ + 無関係2つ |
| 2 | 引用ソースの大半が無関係 | 関連1つ + 無関係3つ |
| 1 | 引用ソースが全て無関係、またはソース引用なし | Vision質問にstreaming引用のみ |

**判定ルール**:
- NotebookLMはソース引用機能がないため、このスコアは **RAGのみ** に適用
- NotebookLMには一律 **N/A** を付与

## 採点手順

1. **期待回答を読む** — 質問が求める情報の範囲を把握
2. **ソース文書を確認** — 期待回答の情報がソースに存在するか確認
3. **回答を読む** — 事実の正しさ、カバー範囲を上記基準で評価
4. **各軸を独立に採点** — 正確性が高くても網羅性が低い場合がある
5. **ソースに情報がない場合の扱い** — 上記の判定ルールに従う
