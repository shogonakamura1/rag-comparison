# 採点ルーブリック v2

## 基本原則

- **誤りは誤り**。軽微・重大の区別はしない。
- **推測は全てNG**。ただし論理的推論は条件付きで許容。
- **網羅性はポイント数の割合で機械的に計算**。

---

## 評価軸 1: 正確性 (Accuracy)

### スコア定義

| スコア | 条件 |
|:------:|------|
| 5 | ソース文書の内容をそのまま正確に述べている |
| 5 | ソースから論理的に導ける推論 + **「これは論理的に導いた答えです」等の明示がある** |
| 5 | ソースに情報がなく「情報がありません」と正直に回答 |
| 4 | ソースの情報を**言い換えて**述べている（意味は正しいが原文と表現が異なる） |
| 3 | ソースから論理的に導ける推論だが、**推論である旨の明示がない** |
| 3 | ソースの情報を言い換えており、かつ一部ニュアンスがずれている |
| 2 | **古い情報**を最新の情報であるかのように述べている |
| 2 | ソースにない情報を**推測で補っている** |
| 1 | 事実として誤った記述がある |
| 1 | ソースに情報があるのに「見つかりませんでした」と回答（検索失敗） |

### 判定フロー

```
1. 回答の各記述がソース文書に存在するか確認
   ├─ 存在する → そのまま引用？ → 5点
   │              言い換え？ → 4点（ニュアンスずれは3点）
   ├─ 存在しないが論理的に導ける
   │   ├─ 「推論です」と明示 → 5点
   │   └─ 明示なし → 3点
   ├─ 存在しない（推測）→ 2点
   ├─ 古い情報を現在形で述べている → 2点
   └─ 事実として誤り → 1点

2. 複数の記述がある場合、最低スコアを採用
```

### 具体例

| 回答内容 | スコア | 理由 |
|---------|:------:|------|
| 「コンテキストウィンドウは1Mトークンです」（ソースに記載あり） | 5 | そのまま正確 |
| 「非常に大きなコンテキストを処理できます」（1Mとは書かず） | 4 | 言い換え（意味は正しい） |
| 「コーディングに優れているため、コードレビューも可能と考えられます。※ソースにはコーディング能力の記載のみで、コードレビューについては論理的に推測した内容です」 | 5 | 論理的推論 + 明示あり |
| 「コードレビューも可能です」（明示なし） | 3 | 論理的推論だが明示なし |
| 「Claude 3.5 Sonnetは最新のモデルです」 | 2 | 古い情報（現在はレガシー） |
| 「Claudeは画像を生成できます」 | 1 | 事実として誤り |

## 評価軸 2: 網羅性 (Coverage)

### 計算方法

```
1. 期待回答からポイント（要点）を抽出する
2. RAG/NLM回答にそのポイントが含まれているか確認
3. 含まれているポイントの正確性を確認
   - 正確に述べている → カウント
   - 推測で述べている（正確性2点以下）→ カウントしない
   - 推論明示なし（正確性3点）→ 0.5としてカウント
4. スコア = カバー率 × 5（四捨五入）
```

### 計算例

期待回答のポイントが4つの場合：

| カバー状況 | カバー率 | スコア |
|-----------|:-------:|:-----:|
| 4/4 正確にカバー | 100% | 5 |
| 3/4 正確にカバー | 75% | 4 |
| 2/4 正確 + 1/4 推論明示なし | 2.5/4 = 62.5% | 3 |
| 2/4 正確にカバー | 50% | 3 |
| 1/4 正確にカバー | 25% | 1 |
| 0/4 | 0% | 1 |
| 「情報がありません」（ソースに情報なし） | N/A | 3 |

**注**: 期待回答にないが回答に含まれる追加情報は、加点も減点もしない。

## 評価軸 3: ソース引用 (Citation)

RAGのみ評価。NotebookLMはN/A。

| スコア | 条件 |
|:------:|------|
| 5 | 引用ソースが全て回答内容と直接関連 |
| 4 | 引用の大半が関連、1つだけ無関係 |
| 3 | 引用の半数が関連 |
| 2 | 引用の大半が無関係 |
| 1 | 引用が全て無関係 |

---

## 採点手順

1. **期待回答のポイントを番号付きで列挙する**（例: P1, P2, P3, P4）
2. **ソース文書を確認**: 各ポイントがソースに存在するか
3. **回答を読み、各ポイントのカバー状況と正確性を表にする**
4. **正確性**: 判定フローに従い、最低スコアを採用
5. **網羅性**: カバー率から機械的に計算
6. **ソース引用**: 引用ソースの関連度を確認

### 採点記録フォーマット

```markdown
#### Q1: [質問]
期待回答ポイント:
- P1: [ポイント1]
- P2: [ポイント2]
- P3: [ポイント3]

| ポイント | RAG | RAG正確性 | NLM | NLM正確性 |
|---------|:---:|:---------:|:---:|:---------:|
| P1      | ○  | 5         | ○  | 5         |
| P2      | ○  | 4(言い換え)| ○  | 5         |
| P3      | ×  | -         | ○  | 5         |

RAG: 正確性=4(P2の言い換え), 網羅性=2/3=67%→3
NLM: 正確性=5, 網羅性=3/3=100%→5
```
