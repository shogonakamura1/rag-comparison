# 採点ルーブリック v3

## 基本原則

- **誤りは誤り**。軽微・重大の区別はしない。
- **推測は全てNG**。論理的推論は条件付きで許容（三段論法テスト）。
- **網羅性は単語レベルのポイントで機械的に計算**。
- **正確性は記述ごとの平均スコア**（最低スコアではない）。
- **評価者が判定に迷った場合は厳しい方に倒す**。

---

## 評価軸 1: 正確性 (Accuracy)

### スコア定義

| スコア | 条件 |
|:------:|------|
| 5 | ソース文書の内容をそのまま正確に述べている |
| 5 | 論理的推論 + 推論である旨を明示 |
| 5 | ソースに情報がなく「情報がありません」と正直に回答 |
| 4 | ソースの情報を言い換えている（意味は正しいが原文と表現が異なる） |
| 3 | 論理的推論だが明示なし |
| 3 | 言い換えでニュアンスがずれている |
| 2 | 古い情報を現在形で述べている |
| 2 | ソースにない情報を推測で補っている |
| 1 | 事実として誤った記述がある |
| 1 | ソースに情報があるのに「見つかりませんでした」と回答 |

### 集計方法

```
1. 回答の各記述（文単位）にスコアを付与
2. 全記述のスコアの平均を算出（小数点第1位まで）
3. 平均スコアを四捨五入して最終スコアとする
```

例: 記述が4つで、スコアが 5, 5, 4, 2 の場合
→ 平均 = (5+5+4+2)/4 = 4.0 → **4点**

### 「論理的推論」と「推測」の判定

**三段論法テスト**: ソースの記述Aから結論Bを導くとき、「Aならば必ずB」と言えるか？

| テスト結果 | 判定 | スコア |
|-----------|------|:------:|
| 「Aならば必ずB」と言える | 論理的推論 | 明示あり→5, なし→3 |
| 「Aならばおそらく/たぶんB」 | **推測（NG）** | 2 |
| 判定に迷う | **推測として扱う** | 2 |

#### 具体例

| 記述 | ソース情報 | 判定 | 理由 |
|------|-----------|------|------|
| 「長い文書を一度に処理できる」 | 「1Mトークンのコンテキストウィンドウ」 | 論理的推論 | 1M入力可能 → 長文処理可能は定義から導ける |
| 「リアルタイム用途に適する」 | 「最も高速なモデル」 | 論理的推論 | 最高速 → 低レイテンシ → リアルタイム向き |
| 「コードレビューもできる」 | 「コーディングに優れている」 | **推測** | コーディング能力 ≠ レビュー能力。別スキル |
| 「大量データ分析に最適」 | 「データ分析に優れている」 | **推測** | 「優れている」から「最適」は飛躍 |

## 評価軸 2: 網羅性 (Coverage)

### ポイントの抽出方法

期待回答（今後は人間が作成した模範回答）から、**意味のある最小単位の単語・短フレーズ**を抽出する。

#### 分割ルール

| ルール | OK | NG |
|--------|----|----|
| 形容詞+名詞は分けてよい | 「高速」「低コスト」→ 2ポイント | |
| 1つの意味をなす複合語は分けない | 「コストパフォーマンス」→ 1ポイント | 「コスト」「パフォーマンス」に分割 |
| 助詞・接続詞は含めない | | 「の」「で」「また」 |
| 固有名詞はそのまま | 「Claude 3 Haiku」→ 1ポイント | |
| 数値+単位はセット | 「200Kトークン」→ 1ポイント | 「200K」「トークン」に分割 |

#### 抽出例

期待回答: 「Claude 3 Haikuは、Claude 3ファミリーの中で最も高速かつ低コストなモデルです。」

ポイント:
- P1: Claude 3 Haiku
- P2: Claude 3ファミリー
- P3: 高速
- P4: 低コスト
- P5: モデル

### 計算方法

```
1. 期待回答からポイントを抽出（P1, P2, ... Pn）
2. 回答内の各ポイントのカバー状況を確認
   - 正確にカバー（正確性4点以上）→ 1.0
   - 推論明示なし（正確性3点）→ 0.5
   - 推測・誤り（正確性2点以下）→ 0.0
   - 回答に含まれない → 0.0
3. カバースコア = Σ(各ポイントのスコア) / n
4. 網羅性 = ceil(カバースコア × 5)  ※最低1点
```

#### 計算例

ポイントが5つ (P1-P5) の場合:

| ポイント | カバー | 正確性 | スコア |
|---------|:------:|:------:|:------:|
| P1: Claude 3 Haiku | ○ | 5 | 1.0 |
| P2: Claude 3ファミリー | ○ | 4(言い換え) | 1.0 |
| P3: 高速 | ○ | 5 | 1.0 |
| P4: 低コスト | × | - | 0.0 |
| P5: モデル | ○ | 5 | 1.0 |

カバースコア = (1.0+1.0+1.0+0.0+1.0)/5 = 0.8
網羅性 = ceil(0.8 × 5) = **4点**

### 特殊ケース

| 状況 | 網羅性スコア |
|------|:----------:|
| ソースに情報がなく「ありません」と回答 | **3** |
| ソースに情報があるが「ありません」と回答 | **1** |

## 評価軸 3: ソース引用 (Citation)

RAGのみ評価。NotebookLMはN/A。

| スコア | 条件 |
|:------:|------|
| 5 | 引用ソースが全て回答内容と直接関連 |
| 4 | 引用の大半が関連、1つだけ無関係 |
| 3 | 引用の半数が関連 |
| 2 | 引用の大半が無関係 |
| 1 | 引用が全て無関係 |

## 採点手順

### Step 1: ポイント抽出
模範回答（人間が作成）から単語レベルのポイントを抽出し、番号を付ける。

### Step 2: 各記述の正確性判定
回答を文単位に分割し、各文の正確性を判定する。

### Step 3: 正確性スコア算出
全文のスコアの平均を四捨五入。

### Step 4: 網羅性スコア算出
ポイントカバー率から機械的に計算。

### Step 5: ソース引用スコア（RAGのみ）

### 採点記録フォーマット

```markdown
#### Q1: [質問]

**模範回答ポイント**: P1:[単語], P2:[単語], P3:[単語], P4:[単語], P5:[単語]

**RAG正確性** (文単位):
| 文 | スコア | 理由 |
|----|:------:|------|
| [文1] | 5 | ソースそのまま |
| [文2] | 4 | 言い換え |
| [文3] | 2 | 推測 |
→ 平均: (5+4+2)/3 = 3.7 → **4点**

**RAG網羅性**:
| ポイント | カバー | 正確性 | スコア |
|---------|:------:|:------:|:------:|
| P1 | ○ | 5 | 1.0 |
| P2 | × | - | 0.0 |
| P3 | ○ | 3 | 0.5 |
→ カバー: 1.5/3 = 0.5 → ceil(0.5×5) = **3点**

**NLM** も同様に採点。
```
